# 原型验证笔记

## 问题
「摸底→计划→打卡→进度→调整 这条状态链路走得通吗？」

## 答案
走得通。核心发现：

1. **休息日必须显式处理**：初始设计中休息日也被纳入打卡计数，导致完成度评估失真。
   修复：休息日自动跳过，打卡只针对训练日。

2. **完成度评估只对跑步类型有效**：力量/拉伸/热身日不应参与完成度百分比计算。
   修复：只有 `run/vo2max/lactate` 类型参与完成度估算。

3. **连续触发机制有效**：3 次低完成打卡正确触发 `consecutive_under=3` 和减量警告。

4. **伤病关键词匹配准确**：「膝盖外侧」正确命中「髂胫束综合征」并分级为 WARN。

## 验证通过的功能
- 摸底评估 5 轮流程
- 模板匹配（5km 目标 → 8 周模板）
- 训练计划生成（含热身/拉伸/力量/VO2max/乳酸模块渐进引入）
- 打卡确认机制（待确认 → 回显 → 确认/取消）
- 伤病预警（关键词 + 分级）
- 动态调整（连续触发 + 三档完成度）
