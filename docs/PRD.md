# 跑步教练智能体 — 产品需求文档 (PRD)

## Problem Statement

跑步新手面临四大痛点：
1. **不知道怎么开始**——网上计划千篇一律，不知道哪个适合自己
2. **容易受伤**——不懂热身、拉伸、力量训练的重要性，也不知道伤痛信号意味着什么
3. **坚持不下去**——没有进度追踪，跑了几周看不到变化就放弃了
4. **遇到问题没人问**——膝盖疼、该穿什么鞋、跑前吃什么，无处咨询

## Solution

一个 AI 跑步教练智能体，通过多轮对话摸底跑者信息，生成涵盖跑步、热身、拉伸、力量、VO2max、乳酸耐受六大模块的个性化训练计划，支持每日打卡追踪与动态调整，内置伤病预警机制，并搭载 RAG 知识库解答跑步相关的各类问题。

## User Stories

1. 作为一个跑步新手，我希望智能体先问我几个关于身体情况和目标的问题，这样它才能给我一份真正适合我的计划，而不是一个通用模板
2. 作为一个跑步新手，我希望得到一份按周排列的训练计划，每天该做什么一目了然，包含跑步、热身、拉伸、力量训练
3. 作为一个想提升能力的跑者，我希望训练计划中能穿插安排间歇跑和节奏跑，帮助我提升最大摄氧量和乳酸耐受度
4. 作为一个跑步新手，我希望每天跑完后能打卡记录距离、配速和体感，智能体会告诉我进度是领先还是落后
5. 作为一个跑步新手，如果我这周完成得特别好或特别差，我希望智能体能自动调整下周的计划
6. 作为一个跑步新手，当我说「膝盖外侧疼」时，我希望智能体不要鼓励我继续跑，而是告诉我可能是什么问题、该不该停跑
7. 作为一个跑步新手，我希望随时问「跑鞋怎么选」「岔气了怎么办」「跑前能喝咖啡吗」并得到专业回答
8. 作为一个跑步新手，我希望训练计划中明确标出哪天是休息日，并且告诉我休息日是训练的一部分而非偷懒
9. 作为一个跑步新手，我希望每次训练前能看到热身动作的具体说明（哪个动作、做多久）
10. 作为一个跑步新手，我希望跑完后有对应的拉伸指导，按部位告诉我该拉哪里、拉多久

## Implementation Decisions

### 技术栈（经设计评审确认）

| 选型 | 决定 |
|------|------|
| 开发框架 | **LangChain + Python** |
| 大模型 | **DeepSeek**（chat 模型） |
| 向量数据库 | **Chroma** |
| 前端 | **Streamlit** |
| 状态持久化 | **自建 JSON 文件** |

### 训练计划生成：模板 + AI 填充

- 预置 4 套 JSON 训练模板：6 周入门 3K / 8 周入门 5K / 10 周入门 10K / 12 周半马入门
- 每个模板的课次包含参数槽位（`{duration}`, `{pace}`），LLM 根据摸底数据填充，不生成结构
- 热身模块、拉伸模块、力量模块在对应训练日自动附加
- VO2max 间歇模块和乳酸耐受模块在后半程（第 4 周起）渐进引入

### 打卡解析：LLM Function Calling + 回显确认

- 用户自由文本输入 → LLM 用 function calling 抽取（距离/配速/体感）
- 回显解析结果给用户确认
- 确认后写入训练记录，更新进度
- API 不可用时降级为手动表单输入

### 伤病预警：关键词 + LLM 双重判断

- 第一层：关键词匹配快速初筛（膝盖外侧 → 髂胫束综合征，跟腱 → 跟腱炎，胫骨前侧 → 胫骨骨膜炎）
- 第二层：LLM 读完整上下文做分级判断
- 三级响应：一级「建议关注」/ 二级「建议减量」/ 三级「建议停跑就医」
- 必须在交互中声明「我不是医生，严重情况请就医」

### 动态调整：三档 + 连续触发

| 完成度 | 连续次数 | 动作 |
|------|------|------|
| <60% | 连续 2 次 | 大幅减量（回退到上上周强度）+ 询问原因 |
| 60%-79% | 连续 2 次 | 微调（下周减 10%） |
| 80%-120% | — | 维持 |
| >120% | 连续 2 次 | 微调（下周加 10%） |
| >150% | 任意 | 警告「加量太快容易受伤，建议控制」 |

### 摸底评估：渐进式 5 轮

1. 欢迎 + 目标 & 截止日期
2. 当前跑步水平（最远距离/配速）
3. 每周可用训练天数
4. 年龄、体重、既往伤病
5. 汇总确认 → 生成计划

### 前端：Streamlit 三栏布局

- **左侧栏**：跑者信息 + 目标摘要（固定）
- **中间主区**：对话窗口
- **右侧栏**：本周课表表格 + 累计进度折线图

### 状态持久化：JSON 文件

- `runners/{id}.json` — 跑者档案
- `plans/{id}.json` — 训练计划
- `progress/{id}.json` — 打卡记录 + 进度

### API 容错

- 超时/503：自动重试 2 次 → 失败返回「教练暂时繁忙」
- 非关键操作降级为表单模式
- 界面显示状态灯：🟢 在线 / 🔴 繁忙

### 知识库来源

- NHS Couch to 5K、Runner's World 新手专栏等公开训练指南
- 丁香医生等公开运动医学科普
- 通用跑步常识（自写 Markdown）
- 每篇文档 ≤ 500 字，分块存入 Chroma

## Testing Decisions

- 准备 5 组不同画像的跑者（不同年龄/体重/目标），逐一验证计划生成质量
- 准备 10 条典型伤痛描述，验证预警分级准确率
- 准备 20 条跑步知识问题，人工评估 RAG 回答的相关性和准确性
- 只测试外部可观察行为，不测试内部实现细节
- 特别关注：伤病预警绝不输出鼓励带伤训练的回复

## Out of Scope

- GPS 轨迹记录与配速自动分析
- 可穿戴设备数据接入
- 社交功能（跑友 PK、分享）
- 多智能体协作
- 实际支付/购买装备
- 视频教学内容
- 移动端原生 App

## Further Notes

- 模板参数调整遵循渐进超负荷原则（周增幅 ≤ 10%）
- 伤病关键词映射表从运动医学公开资料整理
- Streamlit session_state 管单次会话，JSON 文件管跨会话持久化
